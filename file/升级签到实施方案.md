## 教师发布签到与审核签到（含时间/位置/手势序列/明文密码）实施方案

### Summary
- 新增“签到任务”模型，由教师发布并配置约束条件：时间窗口、地理围栏（中心点+半径）、手势序列、明文口令。
- 新增“签到提交与审核”流程：先自动判定，异常进入待审核队列，教师可审核通过/驳回并写回考勤结果。
- 在管理端新增“当前用户展示”：实时展示当前任务已签到用户、异常用户、待审核用户与统计。

### Key Changes
- 后端数据与接口（Workers + D1）
  - 新增表 `CheckinTask`：`id/classId/teacherId/title/startAt/endAt/status/locationLat/locationLng/locationRadiusM/gestureSequence/passwordPlain/createdAt`.
  - 新增表 `CheckinSubmission`：`id/taskId/studentId/submittedAt/lat/lng/gestureInput/passwordInput/autoResult/manualResult/finalResult/reason/isLatest/reviewerId/reviewedAt`.
  - 新增接口（教师端鉴权）：
    - `POST /api/checkin/tasks` 发布签到任务（可直接置为 ACTIVE）。
    - `GET /api/checkin/tasks` 查询任务（按 classId/status）。
    - `POST /api/checkin/tasks/:id/close` 结束任务。
    - `GET /api/checkin/tasks/:id/current-users` 当前用户展示（已签到/待审核/异常/未签到）。
    - `GET /api/checkin/tasks/:id/review-queue` 异常待审核列表。
    - `POST /api/checkin/submissions/:id/review` 教师审核（approve/reject + note）。
  - 新增接口（签到提交入口，供 Android 或管理端模拟）：
    - `POST /api/checkin/tasks/:id/submit` 提交 `studentId + passwordInput + gestureInput + lat/lng`.
  - 判定规则（自动）：
    - 时间：`startAt <= submittedAt <= endAt` 否则异常。
    - 位置：计算与任务中心点距离，超半径异常。
    - 手势：手势序列完全匹配才通过。
    - 密码：明文完全匹配才通过。
    - 全部命中 -> `finalResult=APPROVED`；否则 `finalResult=PENDING_REVIEW` 并记录失败原因。
  - 与现有考勤表联动：
    - 自动通过或教师审核通过后，写入/更新 `AttendanceResult`（默认 `Present`；超时但允许补签时可标 `Late`）。
    - 驳回不写 `Present`，保留审核记录用于追踪。

- 前端（管理端）
  - `Attendance` 页升级为 4 个区域：发布任务、活动任务、待审核队列、当前用户展示。
  - 发布任务表单：
    - 班级、标题、开始/结束时间、地理中心与半径、手势序列（如 `2-4-1`）、明文口令。
    - 约束开关可选（位置/手势/密码），默认全开。
  - 当前用户展示：
    - 实时刷新（轮询）显示：总人数、已签到、待审核、异常、未签到。
    - 列表展示每个学生的最新状态、签到时间、失败原因（若有）。
  - 审核区：
    - 教师可对异常提交执行“通过/驳回”，并可填写审核备注。

- 类型与协议
  - 新增类型：`CheckinTask`, `CheckinSubmission`, `CheckinConstraint`, `ReviewAction`, `CurrentUserItem`.
  - 明确状态枚举：
    - 任务：`DRAFT | ACTIVE | CLOSED`
    - 提交：`APPROVED | PENDING_REVIEW | REJECTED`
    - 自动结果：`PASS | FAIL`

### Test Plan
- 后端接口测试
  - 发布任务参数校验（时间窗口、半径、手势序列格式、明文口令非空）。
  - 提交判定：时间/位置/手势/密码单因子失败与多因子失败组合。
  - 教师隔离：非本人班级任务不可发布、不可审核、不可查看当前用户。
  - 审核幂等：同一提交重复审核时只允许第一次生效或返回明确错误。
- 前端交互测试
  - 发布任务后在活动任务中可见，当前用户统计初始正确。
  - 学生提交后当前用户列表和待审核列表刷新正确。
  - 审核通过后从待审核队列移除并进入已签到统计。
- 回归测试
  - 现有 `GET /api/attendance`、Dashboard 统计不回退。
  - Android 同步相关 `/api/sync/*` 行为保持不变。

### Assumptions
- 本期按你选择采用：`API + 管理端模拟`，不新增学生独立网页。
- 手势签到采用“手势序列”明文比对（例如 `2-4-1`）。
- 密码签到采用“发布时明文口令、提交时明文比对”。
- “当前用户展示”定义为“当前活动签到任务下，按学生聚合后的最新签到状态与明细”。
- 学生身份在 v1 通过 `studentId` 提交（来自可信客户端/管理端模拟）；不做额外学生登录体系。
